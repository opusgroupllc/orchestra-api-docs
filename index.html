    Introduction

    Welcome to the Orchestra API! You can use our API to access API endpoints, which can get information on the various models that power Orchestra.

    This example API documentation page was created with Slate. Feel free to edit it and use it as a base for your own API’s documentation.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "http://localhost:3000/api/v1/users/sessions"
 -d "email=<valid_email>&password=<valid_pwd>"

    Orchestra uses a JSON Web Token to allow access to the API.

    Orchestra expects the JWT to be included in all API requests to the server in a header that looks like the following:

    Authorization: Bearer <valid_json_web_token>

    You must replace `` with your personal API key.

    Users

    Get the Logged in User

    curl "http://example.com/api/v1/users/me"
  -H "Authorization: <valid_json_web_token>"

    The above command returns JSON structured like this:

    {
  "id": 3,
  "email": "test@test.com",
  "username": "test",
  "gravatar": "",
  "full_name": "John Doe",
  "title": "Tester",
  "location": "",
  "github": "",
  "twitter": ""
}

    This endpoint retrieves the currently logged in user.

    HTTP Request

    GET http://example.com/api/v1/users/me/<ID>

    Don’t forget to provide the JWT in the header!

    Get All Users

    curl "http://example.com/api/v1/users"
  -H "Authorization: <valid_json_web_token>"

    The above command returns JSON structured like this:

    [
  {
    "id": 3,
    "email": "test@test.com",
    "username": "test",
    "gravatar": "",
    "full_name": "John Doe",
    "title": "Tester",
    "location": "",
    "github": "",
    "twitter": ""
  }
]

    This endpoint retrieves all users.

    HTTP Request

    GET http://example.com/api/v1/users/:id

    Get a User by ID

    curl "http://example.com/api/v1/users/me"
  -H "Authorization: <valid_json_web_token>"
  -d "id=<valid_id>"

    The above command returns JSON structured like this:

    {
  "id": 3,
  "email": "test@test.com",
  "username": "test",
  "gravatar": "",
  "full_name": "John Doe",
  "title": "Tester",
  "location": "",
  "github": "",
  "twitter": ""
}

    This endpoint retrieves a User by his ID

    HTTP Request

    GET http://example.com/api/v1/users/:id<ID>

    Don’t forget to provide a valid ID in the URL

    Errors

    This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files…just save them to the `includes` folder and add them to the top of your `index.md`’s frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request sucks
    401 Unauthorized – Your API key is wrong
    403 Forbidden – The kitten requested is hidden for administrators only
    404 Not Found – The specified kitten could not be found
    405 Method Not Allowed – You tried to access a kitten with an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The kitten requested has been removed from our servers
    418 I’m a teapot
    429 Too Many Requests – You’re requesting too many kittens! Slown down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarially offline for maintanance. Please try again later.
